# 中医门诊预约小程序 — 完整设计方案

## 一、项目概述

中医门诊预约系统，支持患者在线预约服务项目、技师扫码核销、管理员后台管理营业配置和技师排班。

## 二、系统架构

```
┌──────────────────────────────────┐
│        微信小程序                  │
│  ┌────────────┐ ┌─────────────┐  │
│  │  患者端     │ │  技师端      │  │
│  │ （默认入口）│ │（技师登录后）│  │
│  └────────────┘ └─────────────┘  │
└──────────────┬───────────────────┘
               │
      ┌────────▼────────┐
      │   微信云开发      │
      │  云函数 + 云数据库 │
      │  + 云存储         │
      └────────┬────────┘
               │
      ┌────────▼────────┐
      │  管理后台（H5）   │  ← 部署在云开发静态托管
      │  Vue3 + Element  │
      └─────────────────┘
```

## 三、三种角色

| 角色 | 入口 | 核心能力 |
|------|------|----------|
| **患者** | 微信小程序（默认界面） | 预约、查看预约、出示二维码、取消预约 |
| **技师** | 微信小程序（登录后自动切换技师界面） | 查看所有预约、扫码核销 |
| **管理员** | H5 管理后台（云开发静态托管） | 营业设置、服务管理、技师管理、预约管理、提成统计 |

**角色识别逻辑：**
- 用户打开小程序 → 登录页 → 微信快捷登录（授权获取手机号）
- 用获取到的手机号查询 technicians 表：
  - 匹配到技师 → 进入技师首页
  - 未匹配 → 默认为患者，进入患者首页
- 患者和技师都是同一个微信快捷登录，系统根据手机号自动区分角色

## 四、核心业务流程

### 4.1 患者预约流程

```
微信授权登录
  → 浏览服务项目列表（不显示价格）
  → 选择一个或多个服务项目
  → 选择日期（日历组件，灰掉休班日和过去日期）
  → 系统动态计算可用时段（基于：营业时间 + 服务总时长 + 当天上班技师数 - 已预约数）
  → 选择时段
  → 确认预约
  → 显示"预约成功，请到预约列表查看"
  → 收到预约成功通知
  → 在"我的"→预约列表→预约详情 中查看太阳码
```

### 4.2 技师核销流程

```
技师登录（自动识别为技师身份）
  → 进入技师界面，查看所有预约列表
  → 点击"扫码核销"
  → 扫描患者小程序太阳码
  → 系统验证：预约状态是否为 pending
  → 核销成功，记录核销技师，预约状态变为 completed
  → 患者收到核销完成通知
```

**注意：** 任意技师账号都可以核销任意预约，扫码的技师即为该预约的服务技师。

### 4.3 管理员操作流程

```
登录 H5 管理后台
  → 设置每周各天的营业时间（每天最多 3 个时段）
  → 设置休息管理（店铺停业日 + 技师休假，日历标记 + 法定节假日快捷导入）
  → 添加服务项目（名称、时长、默认提成）
  → 添加技师（姓名、手机号）
  → 设置技师个人提成（可覆盖默认提成）
  → 查看所有预约记录
  → 查看技师提成统计
```

## 六、动态时段计算算法（核心）

```
函数 getAvailableSlots(date, serviceIds):

1. 校验日期
   - 是否在 max_advance_days 范围内
   - 是否非休班日（holidays）
   - 获取该日期是周几 → 查 schedule 获取营业时段

2. 计算服务总时长
   - 遍历 serviceIds，累加每个服务的 duration
   - 如：针灸60分钟 + 推拿30分钟 = 90分钟

3. 获取当天上班技师数
   - 查询 technicians 表 status=active 的技师数
   - 减去 tech_days_off 中该日期的技师数
   - 得到 working_tech_count

4. 获取该日所有已有预约（status = pending）
   - 按 start_time 分组统计每个时段的已预约数

5. 生成候选时段
   - 在每个营业时间段内，按 slot_interval 生成起始时间
   - 如营业段 09:00-12:00，间隔30分钟：09:00, 09:30, 10:00, 10:30, 11:00, 11:30
   - 过滤：start_time + total_duration <= 营业段结束时间

6. 过滤已满时段
   - 对每个候选时段，统计该时段范围内已有的预约数
   - 如果已预约数 >= working_tech_count，则该时段不可用
   - 时段范围：[candidate_start, candidate_start + total_duration)
   - 与已有预约的重叠判断：candidate_start < existing_end AND candidate_end > existing_start

7. 如果是今天，排除已过去的时段

8. 返回可用时段列表
```

## 十一、关键业务规则汇总

| 规则 | 说明 |
|------|------|
| 预约范围 | 可预约未来 1~max_advance_days 天 |
| 每日营业时段 | 最多 3 个，按周几独立设置 |
| 单次预约 | 可选多个服务项目，时长累加 |
| 技师选择 | 患者不选技师，到场后技师扫码即为服务技师 |
| 时段容量 | 当天上班技师数 = 时段可约人数 |
| 技师排班 | 默认全勤，可单独设置某天休假 |
| 休息管理 | 统一管理店铺停业和技师休假，支持日历标记 + 法定节假日快捷导入 |
| 取消预约 | 随时可取消，名额立即释放 |
| 迟到处理 | 不限制，技师随时可扫码核销 |
| 提成规则 | 核销时计算，读取当时的服务价格和提成金额（优先技师个人提成），记录快照，后续修改不影响历史 |
| 价格可见性 | 患者端和技师端不显示价格，仅管理后台可见 |
| 核销规则 | 任意技师可核销任意预约，扫码技师即为服务技师 |
| 黑名单 | 被标记黑名单的客户不可预约，预约时校验 |
| 停业公告 | holidays 中 type="closure" 的日期，患者端首页顶部显示停业公告 + 恢复营业日期 |
| 健康小知识 | 管理端编辑发布，按 sort_order 排序，患者端首页展示卡片列表，点击查看详情 |
